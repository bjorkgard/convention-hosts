## Project Overview

Convention Hosts is a convention management system built with Laravel 12, Inertia.js, and React 19. It manages multi-day events with hierarchical venues (Convention → Floor → Section), real-time occupancy tracking, attendance reporting, and role-based access control.

## Architecture

### Role-Based Access Control

Four-tier role system implemented via pivot tables (not a package like Spatie):
- **Owner** — full control including export and delete
- **ConventionUser** — manage convention, floors, sections, users; lock attendance periods
- **FloorUser** — manage sections on assigned floors
- **SectionUser** — update occupancy and report attendance for assigned sections

Three middleware layers enforce access:
1. `EnsureConventionAccess` — verifies user has any role for the convention
2. `EnsureOwnerRole` — restricts owner-only actions
3. `ScopeByRole` — filters data returned based on role scope

Laravel Policies (`ConventionPolicy`, `FloorPolicy`, `SectionPolicy`, `UserPolicy`) handle action-level authorization. Users can hold multiple roles simultaneously.

### Data Model

```
Convention
├── floors (Floor)
│   └── sections (Section) — occupancy, available_seats, number_of_seats,
│                             elder_friendly, handicap_friendly, information,
│                             last_occupancy_updated_by, last_occupancy_updated_at
├── users (User) — via convention_user pivot; roles in convention_user_roles pivot
│                  User fields: first_name, last_name, email, mobile, email_confirmed
└── attendancePeriods (AttendancePeriod) — max 2 per day (morning/afternoon)
    └── reports (AttendanceReport) — one per section per period
```

Users also have `floor_user` and `section_user` pivots for FloorUser/SectionUser scoping. Note: User has `first_name`/`last_name` separately, not a combined `name` field.

### Occupancy Tracking

`UpdateOccupancyAction` handles three update paths that all normalize to the same `occupancy` (%) and `available_seats` fields:
- Percentage dropdown (0/10/25/50/75/100%)
- "FULL" button (sets to 100%)
- Available-seats input (calculates %, then snaps to nearest dropdown option)

`ResetDailyOccupancy` command runs at 6:00 AM to reset all sections.

### Attendance Reporting Flow

`AttendanceReportService` manages the lifecycle: `startReport()` creates a period, `reportAttendance()` creates/updates per-section reports, `stopReport()` locks the period (immutable after locking). The unique constraint on `(attendance_period_id, section_id)` ensures one report per section per period.

### Export System

`ExportConventionAction` delegates to format-specific exporters (Owner-only):
- `.xlsx` — `ConventionExport` with 4 sheets via maatwebsite/excel
- `.docx` — `ConventionWordExport` via phpoffice/phpword
- `.md` — `ConventionMarkdownExport` plain string building

All exporters eager-load the full relationship tree upfront to prevent N+1.

### Guest Convention Flow

Unauthenticated users can create a convention via `GuestConventionController`. The system emails a signed verification URL. If the user already exists they're auto-logged in; new users set a password via the invitation flow. Unconfirmed guest conventions are cleaned up after 7 days.

### Frontend Structure

- **Pages** — `resources/js/pages/` map 1:1 with backend routes (Inertia)
- **Components** — `resources/js/components/` organized by feature; `ui/` for base primitives (Radix UI, Headless UI)
- **Layouts** — `resources/js/layouts/`
- **Routes** — `resources/js/routes/` and `resources/js/actions/` are **generated by Wayfinder** — do not edit manually
- **Hooks** — `resources/js/hooks/` e.g., `useOccupancyColor`, `useAppearance`
- **Types** — `resources/js/types/`

tests/

│   ├── Auth/             # Auth flows

Use `tests/Helpers/ConventionTestHelper.php` for shared convention/user/role setup before adding duplicate scaffolding to tests.

### Key Patterns

- **Action classes** (`app/Actions/`) — business logic extracted from controllers (e.g., `CreateConventionAction`, `InviteUserAction`, `UpdateOccupancyAction`)
- **Form Requests** — all validation in `app/Http/Requests/`; check sibling files for array vs. string rule style
- **Thin controllers** — delegate to Actions, return Inertia responses
- **Wayfinder** — always import routes from `@/actions/` or `@/routes/` rather than hardcoding URLs in React

### Routes

Defined in `routes/web.php` and `routes/settings.php`. Convention-scoped routes use nested middleware groups (`EnsureConventionAccess` → `ScopeByRole`). Public routes: home, guest convention creation, email verification, invitation activation, `api/version/latest`.

### PWA

Manifest at `public/manifest.json`, service worker at `public/sw.js` (cache-first for static assets). `InstallPrompt` React component handles the `beforeinstallprompt` event with platform-specific instructions.
