## Problem

The application still needs a defined cookie consent flow. This should cover when the banner appears, how the user's choice is stored, and what behavior changes when consent is accepted or declined. The feature surfaced during recent work and should be captured for a dedicated implementation pass.

## Solution

Design and implement a cookie consent system with clear accept and decline actions, persistent storage of the user's decision, and integration points for any non-essential cookies or tracking behavior. Confirm the required UX, storage mechanism, and any compliance requirements before building the final flow.
